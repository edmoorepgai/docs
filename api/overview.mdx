The PG:AI API gives you programmatic access to account intelligence, territory data, contacts, content, and AI assistant capabilities. Use it to pull intelligence into your own tools, automate workflows, or build custom integrations.

## Base URL

```
https://api.getpg.ai/v1
```

## Quick Start

<Steps>
  <Step title="Get your API key">
    Generate an API key in **Configuration > API** within your PG:AI workspace.
  </Step>
  <Step title="Make your first request">
    ```bash
    curl -X GET "https://api.getpg.ai/v1/territories" \
      -H "x-api-key: your_api_key"
    ```
  </Step>
  <Step title="Explore the API Reference">
    Head to the **API Reference** tab to see all endpoints with interactive documentation and try them out.
  </Step>
</Steps>

## Import into Postman

You can import the full PG:AI API specification into Postman or any other API client:

1. Download the [OpenAPI Specification](/openapi.json)
2. In Postman, click **Import** and select the downloaded `openapi.json` file
3. All 26 endpoints will be imported with their parameters, request bodies, and response schemas

<Tip>
The OpenAPI spec is always available at [docs.getpg.ai/openapi.json](https://docs.getpg.ai/openapi.json) and stays in sync with the latest API.
</Tip>

## Rate Limits

API requests are rate-limited based on your plan. When you hit the limit, the API returns `429 Too Many Requests`. See the [Authentication](/api/authentication) page for details on rate limit headers.
